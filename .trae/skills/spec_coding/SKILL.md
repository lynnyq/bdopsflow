## 技能描述

本技能专为 BDopsFlow 分布式运维调度平台的需求开发设计，确保代码质量、安全性和可维护性。

## 适用场景

- 新增功能需求
- 功能优化改进
- 架构设计与重构
- 数据库变更
- 配置项调整

## 前置条件

1. 必须先阅读项目的 [AGENTS.md](AGENTS.md) 文档
2. 了解项目的目录结构和技术栈
3. 明确需求的范围和边界

## 开发流程

### 1. 需求分析

- **理解需求**：仔细阅读用户需求，明确业务场景和目标
- **评估影响**：分析对现有系统的影响范围
- **兼容性检查**：确保新功能向下兼容现有接口和数据结构
- **安全性评估**：评估潜在的安全风险

### 2. 架构设计

- **遵循现有架构**：不主动重构正常工作的代码
- **最小改动原则**：在满足需求的前提下，尽量减小改动范围
- **技术选型**：优先使用项目已有技术栈，新增依赖需评估
- **数据设计**：如需数据库变更，遵循数据库变更规范

### 3. 代码实现

#### 核心规范

1. **错误处理**：所有 error 必须接收校验，禁止使用 `_` 忽略错误
2. **安全规范**：密码、地址、密钥统一从配置/环境变量读取，禁止硬编码
3. **日志规范**：统一使用项目日志组件，禁用 fmt.Println
4. **依赖管理**：新增第三方依赖需要标注用途
5. **性能优化**：高频代码减少临时对象创建

#### 分模块开发

**调度中心开发**
- 在 `scheduler/internal/` 对应目录中实现
- 遵循 handler → service → model 的分层架构
- 新增 API 在 `scheduler/cmd/routes.go` 中注册
- 权限控制使用中间件

**执行器开发**
- 在 `executor/internal/` 对应目录中实现
- gRPC 通信接口定义在 `proto/` 目录
- 任务执行逻辑在 `executor/task_executor.go`

**前端开发**
- Vue 3 Composition API + TypeScript
- 在 `web/src/views/` 中创建页面组件
- API 封装在 `web/src/api/` 中
- 路由配置在 `web/src/router/index.ts`

### 4. 测试

- 新增 Go 测试文件与业务代码同目录
- 新增功能必须包含相应的单元测试
- 运行 `go test ./...` 确保所有测试通过
- 运行前端测试 `cd web && npm test`

### 5. 文档更新

- 更新相关文档（AGENTS.md、README.md 等）
- 数据库变更时更新 `deploy/schema.sql` 和迁移脚本
- 配置变更时更新配置示例文件

### 6. 大改动备注

对于较大的改动，必须备注修改思路：
- 为什么这样设计
- 主要变更点
- 兼容性考虑

## 代码规范检查清单

- [ ] 所有错误都有处理，没有忽略 error
- [ ] 没有硬编码的敏感信息
- [ ] 使用了项目统一的日志组件
- [ ] 新增依赖有明确的用途说明
- [ ] 遵循了最小改动原则
- [ ] 保持了向下兼容性
- [ ] 没有裸 panic
- [ ] 没有跨层直接操作 DB
- [ ] 没有冗余代码
- [ ] 没有过度设计封装

## 常用命令

```bash
# 运行调度中心
cd scheduler && go run cmd/main.go

# 运行执行器
cd executor && go run cmd/main.go --executor-name my-exec --scheduler-addr localhost:50051

# 运行前端
cd web && npm run dev

# 运行 Go 测试
cd scheduler && go test ./internal/... -v

# 格式化代码
gofmt -w .
```
