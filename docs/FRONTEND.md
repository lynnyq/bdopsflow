## 登录与认证

### 登录界面

访问前端地址 `http://localhost:3000`，进入登录界面。

**登录步骤**：

1. 输入用户名
2. 输入密码
3. 点击"登录"按钮

**默认管理员账号**：

- 用户名：`admin`
- 密码：`admin123`

**登录成功**：自动跳转到仪表盘页面。

### 首次登录建议

1. 登录后立即修改默认密码
2. 检查并更新个人信息
3. 了解系统各项功能

## 仪表盘

仪表盘是系统首页，展示系统整体运行状态和关键指标。

### 统计概览

页面顶部显示四组统计卡片：

| 统计项 | 说明 |
|--------|------|
| 总任务 | 系统中所有任务的数量 |
| 已启用 | 当前已启用的定时任务数量 |
| 定时任务 | 配置了 Cron 表达式的任务数量 |
| 运行中 | 当前正在执行的任务数量 |
| 执行器总数 | 注册的执行器总数 |
| 在线执行器 | 当前在线的执行器数量 |
| 工作流总数 | 创建的工作流总数 |
| 已启用工作流 | 当前已启用的工作流数量 |

### 执行统计

显示最近执行的统计信息：

| 统计项 | 说明 |
|--------|------|
| 成功执行 | 最近 N 次执行中成功的次数 |
| 失败执行 | 最近 N 次执行中失败的次数 |
| 平均执行时长 | 所有执行任务的平均耗时 |

### 调度器控制

**调度器状态指示**：

- 绿色脉冲圆点：调度器运行中
- 黄色静止圆点：调度器已暂停

**控制按钮**：

- **恢复调度**：点击后恢复定时任务调度
- **暂停调度**：点击后暂停所有定时任务
- **刷新数据**：手动刷新页面数据

### 执行趋势图

展示最近 7 天的执行趋势：

- 绿色柱状条：成功执行数
- 红色柱状条：失败执行数
- X 轴：日期
- Y 轴：执行数量

**图表说明**：

- 柱状条高度按比例显示
- 鼠标悬停显示具体数值
- 图例说明成功/失败颜色

## 任务管理

### 任务列表

访问路径：侧边栏 → 任务管理

#### 列表页面布局

```
┌──────────────────────────────────────────────────────────┐
│ [+ 创建任务]                              [ID] [名称] [🔍]│
├──────────────────────────────────────────────────────────┤
│ ID │ 名称 │ 类型 │ 状态 │ 领域 │ Cron │ 启用 │ 操作    │
├──────────────────────────────────────────────────────────┤
│ 1  │ API检查│ HTTP │ ●运行│ 领域1 │ */5 * │  ✓  │ ✏️ 🗑️ │
│ 2  │ 备份任务│Shell │ ○待执│ 领域1 │ 0 2 * │  ✓  │ ✏️ 🗑️ │
└──────────────────────────────────────────────────────────┘
```

#### 筛选器

| 筛选字段 | 说明 |
|----------|------|
| ID | 按任务 ID 精确筛选 |
| 名称 | 按任务名称模糊筛选 |
| 类型 | 按任务类型筛选（HTTP/Shell） |
| 状态 | 按任务状态筛选 |
| 领域 | 按所属领域筛选 |

#### 操作列按钮

| 按钮 | 图标 | 功能 |
|------|------|------|
| 编辑 | ✏️ | 打开任务编辑对话框 |
| 删除 | 🗑️ | 删除任务 |
| 触发 | ▶️ | 立即执行任务（手动触发） |
| 查看日志 | 📋 | 查看任务执行日志 |

### 创建任务

点击"创建任务"按钮，弹出任务创建对话框。

#### 基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务名称 | 文本框 | 是 | 任务的显示名称 |
| 任务类型 | 下拉选择 | 是 | HTTP / Shell |
| 所属领域 | 下拉选择 | 是 | 任务归属的领域 |
| Cron 表达式 | 文本框 | 否 | 定时调度规则 |

#### 任务配置

**HTTP 任务配置**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 请求地址 | URL 文本框 | 是 | 完整的 HTTP URL |
| 请求方法 | 下拉选择 | 否 | GET/POST/PUT/DELETE |
| 请求头 | 键值对 | 否 | 自定义 HTTP 头 |
| 请求体 | 文本框 | 否 | JSON 格式请求体 |
| 超时时间 | 数字输入 | 否 | 毫秒，默认 10000 |

**Shell 任务配置**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 脚本内容 | 多行文本框 | 是 | 要执行的 Shell 脚本 |

#### 高级设置

| 字段 | 类型 | 说明 |
|------|------|------|
| 超时时间（秒） | 数字 | 任务执行超时时间，默认 300 |
| 最大重试次数 | 数字 | 失败重试次数，默认 3 |
| 重试间隔（秒） | 数字 | 重试间隔，默认 5 |
| 启用任务 | 开关 | 是否启用定时调度 |
| Webhook 配置 | 文本框/JSON | 回调通知配置 |
| 指定执行器 | 下拉选择 | 固定分配的执行器 |

#### Cron 编辑器

点击"Cron 表达式"输入框，打开 Cron 编辑器组件。

**编辑器功能**：

- 可视化时间选择
- 常用表达式快捷选择
- 实时预览下次执行时间
- 表达式语法检查

**常用快捷选项**：

| 选项 | 表达式 | 说明 |
|------|--------|------|
| 每分钟 | `* * * * *` | 每小时的每分钟 |
| 每5分钟 | `*/5 * * * *` | 每 5 分钟执行一次 |
| 每小时 | `0 * * * *` | 每小时整点执行 |
| 每天凌晨 | `0 0 * * *` | 每天 0 点执行 |
| 每周一 | `0 0 * * 1` | 每周一执行 |
| 每月1号 | `0 0 1 * *` | 每月1号执行 |

### 编辑任务

在任务列表中点击编辑按钮，打开任务编辑对话框。

**编辑注意事项**：

- 可以修改所有字段
- 保存后立即生效
- 如果任务正在运行，部分字段不可修改

### 删除任务

1. 点击删除按钮
2. 系统弹出确认对话框
3. 确认删除后任务及其执行记录一并删除

**删除风险提示**：

- 删除后不可恢复
- 相关执行记录一并删除
- 正在运行的任务会立即失败

### 手动触发任务

1. 点击触发按钮（▶️）
2. 系统立即将任务发送到执行器
3. 页面显示触发成功提示
4. 可在日志页面查看执行结果

## 工作流管理

### 工作流列表

访问路径：侧边栏 → 工作流管理

#### 列表页面布局

```
┌──────────────────────────────────────────────────────────┐
│ [+ 创建工作流]                        [名称] [领域] [🔍] │
├──────────────────────────────────────────────────────────┤
│ ID │ 名称 │ 描述 │ 领域 │ 任务数 │ 启用 │ 操作         │
├──────────────────────────────────────────────────────────┤
│ 1  │ ETL流程│ 数据处│ 领域1 │ 5    │  ✓  │ ✏️ ▶️ 🗑️ │
└──────────────────────────────────────────────────────────┘
```

#### 工作流编辑器

点击"创建工作流"或编辑现有工作流，打开工作流编辑器。

**编辑器布局**：

```
┌────────────────────────────────────────────────────────┐
│ 工作流编辑器                              [保存] [取消] │
├─────────────────────────────┬──────────────────────────┤
│                             │ 节点属性面板             │
│                             │ ┌────────────────────┐  │
│      DAG 画布区域            │ │ 节点名称: task1    │  │
│                             │ │ 节点类型: HTTP     │  │
│    ┌─────┐                 │ │ 请求URL: ...       │  │
│    │task1│ ────────────────│ │ 超时时间: 60s     │  │
│    └─────┘                 │ └────────────────────┘  │
│         │                                       │     │
│         ▼                                       │     │
│    ┌─────┐                 │                     │     │
│    │task2│                 │                     │     │
│    └─────┘                 │                     │     │
│                             │                     │     │
└─────────────────────────────┴──────────────────────────┘
```

**画布功能**：

- 拖拽添加节点
- 连接线建立依赖关系
- 拖拽调整节点位置
- 双击节点编辑属性
- 选中节点按 Delete 删除

**节点类型**：

| 类型 | 说明 |
|------|------|
| HTTP | HTTP 请求任务 |
| Shell | Shell 脚本任务 |

**连接规则**：

- 从上游节点拖动到下游节点
- 上游执行完成后才能执行下游
- 支持多输入多输出
- 不允许循环依赖

### 工作流触发

点击触发按钮，手动执行工作流。

**触发流程**：

1. 系统根据 DAG 拓扑排序确定执行顺序
2. 从入度为 0 的节点开始执行
3. 节点执行完成后触发下游节点
4. 所有节点执行完成视为工作流完成

## 执行器管理

### 执行器列表

访问路径：侧边栏 → 执行器管理

#### 列表页面布局

```
┌──────────────────────────────────────────────────────────┐
│ 执行器列表                              [名称] [状态] [🔍]│
├──────────────────────────────────────────────────────────┤
│ ID │ 名称 │ 执行器ID │ 地址 │ 状态 │ 容量 │ 负载 │ 操作 │
├──────────────────────────────────────────────────────────┤
│ 1  │ Ex-1 │ executor-1│:50051│ ●在线│ 10/10│ 30%│  ⚙️ 🗑️ │
└──────────────────────────────────────────────────────────┘
```

#### 状态指示

| 状态 | 颜色 | 说明 |
|------|------|------|
| 在线 | 绿色 | 执行器正常运行 |
| 离线 | 灰色 | 执行器未连接或心跳超时 |

#### 负载显示

| 负载范围 | 颜色 | 说明 |
|----------|------|------|
| 0-50% | 绿色 | 负载正常 |
| 50-80% | 黄色 | 负载较高 |
| 80-100% | 红色 | 负载饱和 |

#### 操作按钮

| 按钮 | 功能 | 权限要求 |
|------|------|----------|
| 上线 | 标记执行器为在线 | 系统管理员 |
| 下线 | 标记执行器为离线 | 系统管理员 |
| 调整容量 | 修改最大并发数 | 系统管理员 |
| 删除 | 删除执行器记录 | 系统管理员 |

#### 调整容量

1. 点击调整容量按钮
2. 输入新的容量值（正整数）
3. 点击确认保存

**容量说明**：

- 容量 = 执行器最大并发任务数
- 调整后执行器下次心跳时生效
- 负载 = 当前运行任务数 / 容量

## 常见问题

### Q: 任务创建成功但没有执行？

**可能原因**：

1. 任务未启用
2. Cron 表达式格式错误
3. 调度器处于暂停状态
4. 没有在线执行器

**排查步骤**：

1. 检查任务是否启用
2. 验证 Cron 表达式
3. 检查调度器状态
4. 确认执行器在线

### Q: 执行失败如何排查？

**排查步骤**：

1. 查看执行日志详情
2. 检查任务配置是否正确
3. 验证目标服务可访问
4. 查看执行器日志

### Q: 如何导出执行记录？

当前版本不支持直接导出。

**替代方案**：

1. 使用 API 获取数据
2. 查看日志页面筛选结果
3. 截图保存关键信息

### Q: 批量操作失败怎么办？

**处理方式**：

1. 查看失败提示信息
2. 逐个检查选中的记录
3. 确保操作权限充足
4. 重新尝试失败项
